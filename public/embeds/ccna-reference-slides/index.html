LAYER 2

Spanning Tree Protocol (STP)

Preventing Loops · Ensuring Stability · IEEE 802.1D

WHAT IS STP?
  • A Layer 2 protocol that prevents loops in switched Ethernet networks
  • Creates a loop-free logical topology by selectively blocking redundant links
  • Defined in IEEE 802.1D — runs automatically on all switches
HOW STP WORKS
1
Elect Root Bridge
All switches exchange BPDUs — lowest Bridge ID wins
2
Select Root Ports
Each non-root switch selects 1 RP — lowest path cost to root
3
Select Designated Ports
Each segment has 1 DP — best path back to root bridge
4
Block Redundant Ports
All remaining ports enter Blocking state — loop eliminated
THE PROBLEM: LAYER 2 LOOPS

Without STP, loops cause:

  • Broadcast storms (frames loop forever)
  • MAC address table instability
  • Duplicate frames delivered to hosts
  • High CPU utilization on all switches
  • Complete network outage
STP TOPOLOGY EXAMPLE
● RP – Root Port ● DP – Designated Port ✕ Blocked Port
STP PORT STATES
Blocking — no frames, no MAC learning
Listening — processes BPDUs, no forwarding
Learning — learns MACs, no forwarding
Forwarding — full operation
Default Timers: Hello 2s · Max Age 20s · Forward Delay 15s
BPDU INFORMATION

BPDUs contain:

✓ Root Bridge ID✓ Root Path Cost ✓ Sender Bridge ID✓ Port ID ✓ Timer values
BRIDGE ID PRIORITY

Bridge ID = Priority + MAC Address

✓ Lower value = Higher priority
✓ Default: 32768 · Range: 0–61440 (steps of 4096)
✓ Tie-break: lowest MAC address wins
STP VARIANTS
802.1D STP — Legacy standard, slow convergence
802.1w RSTP — Rapid STP, faster convergence
802.1s MSTP — Multiple spanning trees per VLAN
PVST+ / Rapid PVST+ — Cisco proprietary
KEY

STP automatically creates a loop-free topology by blocking redundant links while keeping backup paths available — ensuring network stability and preventing broadcast storms.

WHAT IS OSI?

OSI stands for Open Systems Interconnection — a 7-layer reference model developed by ISO to standardize communication between systems.

It divides network communication into 7 layers, each with specific functions and responsibilities.

KEY FEATURES

✓Easy troubleshooting by isolating layers

802.1Q

Native VLAN — Default VLAN for Untagged Traffic

The Default VLAN for Untagged Traffic on Trunk Links

TRUNK LINK DIAGRAM
KEY CONCEPTS
  • Native VLAN traffic is sent untagged over the trunk
  • All other VLANs are tagged using 802.1Q
  • Both ends of the trunk must have the same native VLAN
  • Default native VLAN is VLAN 1 (change for security)
CONFIGURATION EXAMPLE
SWITCH 1
Switch1# configure terminal
Switch1(config)# interface g0/1
Switch1(config-if)# switchport mode trunk
Switch1(config-if)# switchport trunk native vlan 99
Switch1(config-if)# end
SWITCH 2
Switch2# configure terminal
Switch2(config)# interface g0/1
Switch2(config-if)# switchport mode trunk
Switch2(config-if)# switchport trunk native vlan 99
Switch2(config-if)# end
BEST PRACTICE

Use an unused VLAN as the Native VLAN (not VLAN 1) to improve security and prevent VLAN hopping attacks.

IP MASKING

Subnet Mask vs Wildcard Mask

Understanding the key differences and use cases

SUBNET MASK
  • Defines network and host portion of an IP address
  • 1 = Network bit, 0 = Host bit
  • Used in IP addressing and subnetting
  • Applied directly to IP interfaces
EXAMPLE: /24
Subnet Mask: 255.255.255.0
Binary: 11111111.11111111.11111111.00000000
COMMON SUBNET MASKS
/8255.0.0.0 /16255.255.0.0 /24255.255.255.0 /25255.255.255.128 /26255.255.255.192 /27255.255.255.224 /28255.255.255.240 /30255.255.255.252
WILDCARD MASK
  • The inverse of the subnet mask
  • 0 = Must match, 1 = Ignore (wildcard)
  • Used in ACLs and routing protocols (OSPF, EIGRP)
  • Cisco uses it to match IP address ranges
FORMULA
Wildcard = 255.255.255.255 − Subnet Mask
EXAMPLE: /24 → Wildcard
Subnet: 11111111.11111111.11111111.00000000
Wildcard: 00000000.00000000.00000000.11111111
Result: 0.0.0.255
WILDCARD EXAMPLES (ACL USE)
0.0.0.0/32Match exact host 0.0.0.255/24Match entire /24 subnet 0.0.255.255/16Match entire /16 range 255.255.255.255Match any IP address
