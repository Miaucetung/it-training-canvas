TRUNK-LINK · TAGGED vs UNTAGGED
KERNKONZEPTE
  • Native-VLAN-Verkehr läuft untagged über den Trunk
  • Alle anderen VLANs werden via 802.1Q getaggt
  • Beide Trunk-Enden brauchen das gleiche Native VLAN
  • Default ist VLAN 1 — aus Sicherheitsgründen ändern
  • Mismatch → VLAN-Hopping-Angriff möglich
KONFIGURATION · BEIDE SWITCHES
SWITCH 1
interface g0/1
switchport mode trunk
switchport trunk
native vlan 99
switchport nonegotiate
SWITCH 2
interface g0/1
switchport mode trunk
switchport trunk
native vlan 99
switchport nonegotiate
BEST PRACTICE: Ein unbenutztes VLAN als Native nutzen (z.B. 99 statt VLAN 1) → schützt vor VLAN-Hopping.
